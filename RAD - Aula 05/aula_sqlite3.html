01 · Fundamentos

O que é SQLite3?

SQLite é um banco de dados relacional embutido — ele vive dentro de um único arquivo .db no seu disco e não precisa de nenhum servidor rodando em background. O módulo sqlite3 já vem na biblioteca padrão do Python, sem instalar nada.

Diferente de bancos como MySQL ou PostgreSQL, o SQLite não tem processo separado, não exige configuração de usuário/senha e funciona em qualquer sistema operacional. Isso o torna ideal para protótipos, aplicações embarcadas, scripts de análise de dados e ensino.

💡

Nas aulas anteriores usamos o Pandas para ler e manipular CSVs na memória. Com o SQLite3, os dados ficam salvos em disco de forma estruturada e consultável com SQL — e você pode combinar os dois livremente.

🏗 Como o SQLite funciona por dentro

Todo banco SQLite é um único arquivo binário com extensão .db (ou .sqlite). Dentro dele existem páginas de 4KB organizadas em estrutura B-tree. Você não precisa saber disso para usar, mas entender a estrutura ajuda a tomar decisões.

📐 Tipos de dados do SQLite — Dynamic Typing

O SQLite usa um sistema chamado type affinity: diferente do SQL tradicional, ele é flexível quanto ao tipo real armazenado. Na prática você declara os tipos, mas o banco aceita variações.

Tipo SQLiteEquivalente PythonExemplosObservação
INTEGERint1, -5, 2024Armazenado em 1 a 8 bytes
REALfloat3.14, -0.5Ponto flutuante IEEE 754
TEXTstr"Ana", "SP"UTF-8 por padrão
BLOBbytesb"\x89PNG..."Dados binários sem conversão
NULLNoneNULLAusência de valor
⚠️

SQLite não tem tipo DATE nativo. Datas são armazenadas como TEXT ("2024-01-10"), INTEGER (Unix timestamp) ou REAL (Julianday). O Pandas lida com isso automaticamente ao usar pd.to_datetime() após a leitura.

🔒 Constraints — garantindo a qualidade dos dados

Constraints são regras aplicadas diretamente no banco que impedem dados inválidos de entrar, mesmo que o código Python não valide.

constraints.sql
-- NOT NULL: campo obrigatório
-- UNIQUE: valor não se repete na tabela
-- DEFAULT: valor padrão se não informado
-- CHECK: validação customizada
-- FOREIGN KEY: integridade referencial

CREATE TABLE funcionarios (
    id               INTEGER  PRIMARY KEY AUTOINCREMENT,
    nome             TEXT     NOT NULL,
    idade            REAL     CHECK(idade > 0 AND idade < 120),
    cidade           TEXT     DEFAULT 'Não informada',
    salario          REAL     CHECK(salario >= 0),
    data_contratacao TEXT
);

CREATE TABLE livros (
    id      INTEGER  PRIMARY KEY AUTOINCREMENT,
    titulo  TEXT     NOT NULL,
    autor   TEXT,
    isbn    TEXT     UNIQUE,   -- ISBN não pode se repetir
    paginas INTEGER  CHECK(paginas > 0),
    ano     INTEGER  CHECK(ano > 1800 AND ano <= 2100)
);

CREATE TABLE emprestimos (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    livro_id       INTEGER NOT NULL REFERENCES livros(id),
    funcionario_id INTEGER NOT NULL REFERENCES funcionarios(id),
    data_retirada  TEXT    NOT NULL,
    data_devolucao TEXT    -- NULL = ainda emprestado
);

🔍 Inspecionando o banco — sqlite_master

O SQLite guarda informações sobre suas próprias tabelas numa tabela especial chamada sqlite_master. Útil para debug e introspecção.

inspecao.py
import sqlite3

conn   = sqlite3.connect("biblioteca.db")
cursor = conn.cursor()

# Lista todas as tabelas do banco
cursor.execute("SELECT name FROM sqlite_master WHERE type='table'")
tabelas = cursor.fetchall()
print("Tabelas:", tabelas)

# Ver o SQL de criação de uma tabela
cursor.execute("SELECT sql FROM sqlite_master WHERE name='livros'")
print(cursor.fetchone()[0])

# Descrição das colunas de uma tabela
cursor.execute("PRAGMA table_info(funcionarios)")
for col in cursor.fetchall():
    # (cid, name, type, notnull, dflt_value, pk)
    print(f"Coluna: {col[1]:20} | Tipo: {col[2]:10} | NOT NULL: {bool(col[3])}")

# Contar registros em cada tabela
for (nome,) in tabelas:
    cursor.execute(f"SELECT COUNT(*) FROM {nome}")
    qtd = cursor.fetchone()[0]
    print(f"  {nome}: {qtd} registros")

conn.close()

⚙️ PRAGMAs — configurando o comportamento do banco

PRAGMAs são comandos especiais do SQLite para configurar o banco. Os mais importantes:

pragmas.py
# Ativar chaves estrangeiras (DESATIVADO por padrão!)
cursor.execute("PRAGMA foreign_keys = ON")

# Melhorar performance com WAL mode (Write-Ahead Logging)
cursor.execute("PRAGMA journal_mode = WAL")

# Ver tamanho do banco em páginas
cursor.execute("PRAGMA page_count")
print("Páginas:", cursor.fetchone()[0])

# Ver versão do SQLite
print("SQLite versão:", sqlite3.sqlite_version)
print("Módulo versão: ", sqlite3.version)
✅

PRAGMA foreign_keys = ON deve ser executado sempre ao abrir uma conexão. Por padrão, o SQLite ignora violações de chave estrangeira — ativá-lo garante a integridade referencial entre tabelas.

02 · Primeiros Passos

Conectando e Criando Tabelas

O fluxo básico do SQLite3 tem sempre três atores: a conexão, o cursor e os commits.

sqlite3_basico.py
import sqlite3

# Conecta ao banco (cria o arquivo se não existir)
conn = sqlite3.connect("biblioteca.db")

# Cursor é o "executor" dos comandos SQL
cursor = conn.cursor()

# Cria tabela de funcionários (igual ao DataFrame das aulas anteriores!)
cursor.execute("""
    CREATE TABLE IF NOT EXISTS funcionarios (
        id        INTEGER PRIMARY KEY AUTOINCREMENT,
        nome      TEXT    NOT NULL,
        idade     REAL,
        cidade    TEXT,
        salario   REAL,
        data_contratacao TEXT
    )
""")

# Salva as mudanças no disco
conn.commit()

print("Banco criado com sucesso! 🗄")
✅

PRIMARY KEY AUTOINCREMENT — o SQLite gera o id automaticamente a cada linha inserida, sem você precisar controlar. O IF NOT EXISTS evita erros se rodar o script mais de uma vez.

03 · Operações Essenciais

CRUD — Create, Read, Update, Delete

C INSERT — Inserindo dados

Usamos parâmetros com ? para proteger contra SQL Injection. Nunca concatene strings diretamente!

insert.py
# ── Inserir um único registro ──
cursor.execute("""
    INSERT INTO funcionarios (nome, idade, cidade, salario, data_contratacao)
    VALUES (?, ?, ?, ?, ?)
""", ("Ana", 23, "SP", 3000.0, "2020-01-10"))

# ── Inserir vários de uma vez com executemany ──
funcionarios = [
    ("Bruno",   35, "RJ", 5000.0, "2019-03-15"),
    ("Carlos",  29, "SP", 4000.0, "2021-07-22"),
    ("Daniela", None, "MG", 3500.0, "2018-11-30"),
    ("Eduardo", 40, "RJ", None,  "2022-05-10"),
]

cursor.executemany("""
    INSERT INTO funcionarios (nome, idade, cidade, salario, data_contratacao)
    VALUES (?, ?, ?, ?, ?)
""", funcionarios)

conn.commit()
print(f"Linhas inseridas: {cursor.rowcount}")
⚠️

Sempre use ? como placeholder. Nunca faça: f"INSERT ... VALUES ('{nome}')" — isso abre brechas de segurança graves.

R SELECT — Consultando dados

select.py
# fetchall() → lista de tuplas
cursor.execute("SELECT * FROM funcionarios")
todos = cursor.fetchall()
for linha in todos:
    print(linha)

# fetchone() → apenas a primeira linha
cursor.execute("SELECT nome, salario FROM funcionarios WHERE cidade = ?", ("SP",))
primeiro = cursor.fetchone()
print(f"Primeiro de SP: {primeiro}")

# Filtros, ordenação e limite
cursor.execute("""
    SELECT nome, salario
    FROM funcionarios
    WHERE salario > 3500
    ORDER BY salario DESC
    LIMIT 3
""")
print(cursor.fetchall())

# Contagem e agregação
cursor.execute("""
    SELECT cidade, COUNT(*) AS total, AVG(salario) AS media_salarial
    FROM funcionarios
    GROUP BY cidade
    ORDER BY media_salarial DESC
""")
for row in cursor.fetchall():
    print(f"Cidade: {row[0]} | Total: {row[1]} | Média: R$ {row[2]:.2f}")

U UPDATE — Atualizando dados

update.py
# Atualizar salário de um funcionário específico
cursor.execute("""
    UPDATE funcionarios

""", (5500.0, "Bruno"))

# Atualizar com lógica condicional (equivalente ao .apply() do Pandas)
cursor.execute("""
    UPDATE funcionarios
    SET salario = salario * 1.1

🔒 Usando Context Manager (modo recomendado)

O bloco with faz o commit automaticamente e reverte em caso de erro — mais seguro e elegante.

import sqlite3

with sqlite3.connect("biblioteca.db") as conn:
    cursor = conn.cursor()

        INSERT INTO funcionarios (nome, idade, cidade, salario, data_contratacao)
        VALUES (?, ?, ?, ?, ?)
    """, ("Fernanda", 28, "SP", 4200.0, "2023-01-15"))
    # commit() automático ao sair do bloco with
    # rollback() automático se houver exceção

04 · Integração com Pandas

Pandas + SQLite3 — usando os dados das aulas anteriores

Nas aulas anteriores construímos um DataFrame de funcionários com Ana, Bruno, Carlos, Daniela e Eduardo, aplicamos limpeza de dados com fillna(), criamos colunas derivadas, fizemos groupby() e merge. Agora vamos persistir tudo isso no SQLite e combinar com o livros.csv.

🔁

O Pandas tem duas funções-chave para trabalhar com bancos SQL: DataFrame.to_sql() para gravar dados no banco, e pd.read_sql_query() para ler qualquer query SQL de volta como DataFrame. Simples assim.

1 Recriando o DataFrame da Aula 01 e salvando no SQLite

Exatamente os mesmos dados que usamos no notebook PandasPratica.ipynb — agora sendo persistidos num banco real.

aula01_para_sqlite.py
import pandas as pd
import numpy as np
import sqlite3

# ══════════════════════════════════════════════
#  DADOS DA AULA 01 — exatamente como no notebook
# ══════════════════════════════════════════════
dados = {
    "nome": ["Ana", "Bruno", "Carlos", "Daniela", "Eduardo"],
    "idade": [23, 35, 29, np.nan, 40],
    "cidade": ["SP", "RJ", "SP", "MG", "RJ"],
    "salario": [3000, 5000, 4000, 3500, np.nan],
    "data_contratacao": pd.to_datetime([
        "2020-01-10", "2019-03-15", "2021-07-22", "2018-11-30", "2022-05-10"
    ])
}
df = pd.DataFrame(dados)

# ══════════════════════════════════════════════
#  LIMPEZA — igual ao que fizemos na Aula 01
# ══════════════════════════════════════════════
df["idade"]   = df["idade"].fillna(df["idade"].mean())       # Daniela → 31.75
df["salario"] = df["salario"].fillna(df["salario"].median()) # Eduardo → 3750.0

# Colunas derivadas da Aula 01
df["salario_anual"]    = df["salario"] * 12
df["ano_contratacao"]  = df["data_contratacao"].dt.year
df["categoria_salario"] = df["salario"].apply(
    lambda x: "Alto" if x > 4500 else "Médio" if x > 3000 else "Baixo"
)

# Converte data para string antes de salvar (SQLite não tem tipo DATE)
df["data_contratacao"] = df["data_contratacao"].astype(str)

print(df)
#        nome  idade cidade  salario data_contratacao  salario_anual  ano_contratacao categoria_salario
# 0       Ana  23.00     SP   3000.0       2020-01-10        36000.0             2020             Baixo
# 1     Bruno  35.00     RJ   5000.0       2019-03-15        60000.0             2019              Alto
# 2    Carlos  29.00     SP   4000.0       2021-07-22        48000.0             2021             Médio
# 3   Daniela  31.75     MG   3500.0       2018-11-30        42000.0             2018             Médio
# 4   Eduardo  40.00     RJ   3750.0       2022-05-10        45000.0             2022             Médio

# ══════════════════════════════════════════════
#  PERSISTE NO SQLITE
# ══════════════════════════════════════════════
with sqlite3.connect("biblioteca.db") as conn:
    df.to_sql(
        name="funcionarios",
        con=conn,
        if_exists="replace",  # recria se já existir
        index=False           # não salva o índice do pandas
    )
    print(f"\n✅ {len(df)} funcionários salvos no banco.")

2 Importando o livros.csv da Aula 02

O livros.csv tem separador ;, contém entradas com paginas = 0 (dados inválidos) e duplicatas por ISBN. Vamos aplicar a mesma limpeza da Aula 02 antes de inserir no banco.

livros_para_sqlite.py
import pandas as pd
import sqlite3

# ── Leitura ──
df_livros = pd.read_csv("livros.csv", sep=";")
print(f"Antes da limpeza: {len(df_livros)} registros")
print(df_livros.head())
#                           titulo          autor           isbn  paginas   ano
# 0         Orçamento sem falhas  Nath Finanças  9786555601565      128  2021
# 1       Minha Sombria Vanessa  Kate Elizabeth  9788551006276      432  2020
# ...

# ── Limpeza ──
# Remove livros com 0 páginas (entrada inválida)
df_livros = df_livros[df_livros["paginas"] > 0]

# Remove duplicatas mantendo a primeira ocorrência por ISBN
df_livros = df_livros.drop_duplicates(subset="isbn", keep="first")

# Remove espaços extras nos títulos (ex: "Uma chance de lutar; Elizabeth Warren")
df_livros["titulo"] = df_livros["titulo"].str.strip()
df_livros["autor"]  = df_livros["autor"].str.strip()

# Colunas derivadas úteis
df_livros["categoria_tamanho"] = pd.cut(
    df_livros["paginas"],
    bins=[0, 150, 300, 500, 9999],
    labels=["Curto", "Médio", "Longo", "Extenso"]
)

print(f"Após limpeza: {len(df_livros)} registros")
print(df_livros.describe())

# ── Persiste no SQLite ──
with sqlite3.connect("biblioteca.db") as conn:
    df_livros.to_sql("livros", conn, if_exists="replace", index=False)
    print(f"\n✅ {len(df_livros)} livros importados para o banco.")

3 Consultando dados com SQL e analisando com Pandas

Com pd.read_sql_query() qualquer query SQL vira um DataFrame. Você usa SQL para filtrar e agregar no banco, e Pandas para análise posterior.

consultas_pandas.py
import pandas as pd
import sqlite3

conn = sqlite3.connect("biblioteca.db")

# ── Equivalente ao groupby("cidade")["salario"].mean() da Aula 01 ──
df_cidades = pd.read_sql_query("""
    SELECT
        cidade,
        COUNT(*)        AS total_funcionarios,
        ROUND(AVG(salario), 2)  AS media_salarial,
        MIN(salario)    AS menor_salario,
        MAX(salario)    AS maior_salario
    FROM funcionarios
    GROUP BY cidade
    ORDER BY media_salarial DESC
""", conn)
print("Média salarial por cidade:")
print(df_cidades)
#   cidade  total_funcionarios  media_salarial  menor_salario  maior_salario
# 0     RJ                   2         4375.00         3750.0         5000.0
# 1     SP                   2         3500.00         3000.0         4000.0
# 2     MG                   1         3500.00         3500.0         3500.0

# ── Funcionários com salário acima da média — equivalente ao filtro df[df.salario > x] ──
media = pd.read_sql_query("SELECT AVG(salario) AS media FROM funcionarios", conn)["media"][0]

df_acima = pd.read_sql_query("""
    SELECT nome, cidade, salario, categoria_salario
    FROM funcionarios
    WHERE salario > ?
    ORDER BY salario DESC
""", conn, params=(media,))
print(f"\nAcima da média (R${media:.2f}):")
print(df_acima)

# ── Análise de livros por ano — igual ao groupby da Aula 02 ──
df_anos = pd.read_sql_query("""
    SELECT
        ano,
        COUNT(*)          AS total_livros,
        ROUND(AVG(paginas)) AS media_paginas,
        SUM(paginas)      AS total_paginas
    FROM livros
    GROUP BY ano
    ORDER BY ano DESC
""", conn)
print("\nPublicações por ano:")
print(df_anos.head(10))

# ── Top autores (equivalente ao value_counts()) ──
df_autores = pd.read_sql_query("""
    SELECT autor, COUNT(*) AS total, SUM(paginas) AS total_paginas
    FROM livros
    WHERE autor IS NOT NULL AND autor != ''
    GROUP BY autor
    HAVING COUNT(*) > 1
    ORDER BY total DESC
    LIMIT 10
""", conn)
print("\nAutores com mais de 1 livro:")
print(df_autores)

conn.close()

4 JOIN no SQL vs merge() no Pandas — lado a lado

Na Aula 01 usamos pd.merge(df, bonus, on="nome"). Agora fazemos o mesmo conceito diretamente no SQL, e depois voltamos para Pandas para continuar a análise.

join_comparado.py
import pandas as pd
import sqlite3

conn   = sqlite3.connect("biblioteca.db")
cursor = conn.cursor()

# ── Cria tabela de bônus (igual ao DataFrame "bonus" da Aula 01) ──
cursor.execute("""
    CREATE TABLE IF NOT EXISTS bonus (
        nome  TEXT PRIMARY KEY,
        bonus INTEGER
    )
""")
cursor.executemany(
    "INSERT OR REPLACE INTO bonus VALUES (?, ?)",
    [("Ana", 500), ("Bruno", 1000), ("Carlos", 700),
     ("Daniela", 600), ("Eduardo", 800)]
)
conn.commit()

# ════ ABORDAGEM 1: pd.merge() — jeito Pandas ════
df_func  = pd.read_sql_query("SELECT * FROM funcionarios", conn)
df_bonus = pd.read_sql_query("SELECT * FROM bonus", conn)
df_merge_pandas = pd.merge(df_func, df_bonus, on="nome")
df_merge_pandas["total_anual"] = df_merge_pandas["salario_anual"] + df_merge_pandas["bonus"]
print("Merge Pandas:\n", df_merge_pandas[["nome", "salario", "bonus", "total_anual"]])

# ════ ABORDAGEM 2: SQL JOIN — jeito SQLite ════
df_merge_sql = pd.read_sql_query("""
    SELECT
        f.nome,
        f.cidade,
        f.salario,
        f.salario_anual,
        f.categoria_salario,
        b.bonus,
        f.salario_anual + b.bonus AS total_anual
    FROM funcionarios f
    INNER JOIN bonus b ON f.nome = b.nome
    ORDER BY total_anual DESC
""", conn)
print("\nJOIN SQL:\n", df_merge_sql)
#       nome cidade  salario  salario_anual categoria_salario  bonus  total_anual
# 0    Bruno     RJ   5000.0        60000.0              Alto   1000      61000.0
# 1   Carlos     SP   4000.0        48000.0             Médio    700      48700.0
# 2  Eduardo     RJ   3750.0        45000.0             Médio    800      45800.0
# 3  Daniela     MG   3500.0        42000.0             Médio    600      42600.0
# 4      Ana     SP   3000.0        36000.0             Baixo    500      36500.0

# Continua no Pandas normalmente
print("\nMédia de total_anual por cidade:")
print(df_merge_sql.groupby("cidade")["total_anual"].mean().round(2))

conn.close()

5 Pivot Table: Pandas vs SQL

Na Aula 01 usamos pd.pivot_table(). O SQL equivalente usa CASE WHEN para pivotar colunas — mais verboso, mas roda direto no banco sem carregar tudo na memória.

pivot_comparado.py
import pandas as pd
import sqlite3

conn = sqlite3.connect("biblioteca.db")
df   = pd.read_sql_query("SELECT * FROM funcionarios", conn)

# ════ ABORDAGEM 1: pd.pivot_table() ════
pivot_pandas = pd.pivot_table(
    df,
    values="salario",
    index="cidade",
    columns="categoria_salario",
    aggfunc="mean"
)
print("Pivot Pandas:\n", pivot_pandas)
# categoria_salario    Alto   Baixo   Médio
# cidade
# MG                    NaN     NaN  3500.0
# RJ                 5000.0     NaN  3750.0
# SP                    NaN  3000.0  4000.0

# ════ ABORDAGEM 2: CASE WHEN no SQL ════
df_pivot_sql = pd.read_sql_query("""
    SELECT
        cidade,
        AVG(CASE WHEN categoria_salario = 'Alto'  THEN salario END) AS Alto,
        AVG(CASE WHEN categoria_salario = 'Médio' THEN salario END) AS Medio,
        AVG(CASE WHEN categoria_salario = 'Baixo' THEN salario END) AS Baixo
    FROM funcionarios
    GROUP BY cidade
    ORDER BY cidade
""", conn)
print("\nPivot SQL:\n", df_pivot_sql)

conn.close()

        f.salario, f.salario_anual,
        f.categoria_salario, f.data_contratacao,
        f.ano_contratacao,
        b.bonus,
        f.salario_anual + b.bonus AS total_com_bonus
    FROM funcionarios f
    LEFT JOIN bonus b ON f.nome = b.nome
    ORDER BY total_com_bonus DESC
""", conn)

# ── Estatísticas dos livros ──
df_livros_stats = pd.read_sql_query("""
    SELECT ano, COUNT(*) as total, AVG(paginas) as media_pag
    FROM livros GROUP BY ano ORDER BY ano DESC
""", conn)

# ── Exporta para Excel com múltiplas abas ──
with pd.ExcelWriter("resultado_completo.xlsx", engine="openpyxl") as writer:
    df_result.to_excel(writer, sheet_name="Funcionarios", index=False)
    df_livros_stats.to_excel(writer, sheet_name="Livros_por_Ano", index=False)

print("✅ Exportado: resultado_completo.xlsx")
conn.close()
🔁

Fluxo completo da série: CSV → Pandas (limpeza, Aula 01/02) → SQLite (persistência) → SQL (consulta) → Pandas (análise) → Excel (entrega). Cada ferramenta no que faz melhor.

05 · Projeto Completo

Projeto: Sistema Completo da Biblioteca

Junta tudo: dados de funcionários da Aula 01, livros.csv da Aula 02, empréstimos com JOIN, e exportação final. Um script do zero ao fim.

sistema_biblioteca.py
import sqlite3
import pandas as pd
import numpy as np

# ════════════════════════════════════════
#  SETUP
# ════════════════════════════════════════
conn   = sqlite3.connect("biblioteca.db")
cursor = conn.cursor()
cursor.execute("PRAGMA foreign_keys = ON")

cursor.executescript("""
    DROP TABLE IF EXISTS emprestimos;
    DROP TABLE IF EXISTS bonus;
    DROP TABLE IF EXISTS livros;
    DROP TABLE IF EXISTS funcionarios;

    CREATE TABLE funcionarios (
        id               INTEGER PRIMARY KEY AUTOINCREMENT,
        nome             TEXT    NOT NULL,
        idade            REAL,
        cidade           TEXT,
        salario          REAL    CHECK(salario >= 0),
        salario_anual    REAL,
        data_contratacao TEXT,
        ano_contratacao  INTEGER,
        categoria_salario TEXT
    );

    CREATE TABLE livros (
        id       INTEGER PRIMARY KEY AUTOINCREMENT,
        titulo   TEXT NOT NULL,
        autor    TEXT,
        isbn     TEXT UNIQUE,
        paginas  INTEGER CHECK(paginas > 0),
        ano      INTEGER
    );

    CREATE TABLE bonus (
        nome  TEXT PRIMARY KEY,
        bonus INTEGER
    );

    CREATE TABLE emprestimos (
        id             INTEGER PRIMARY KEY AUTOINCREMENT,
        livro_id       INTEGER NOT NULL REFERENCES livros(id),
        funcionario_id INTEGER NOT NULL REFERENCES funcionarios(id),
        data_retirada  TEXT    NOT NULL,
        data_devolucao TEXT
    );
""")
conn.commit()
print("✅ Tabelas criadas")

# ════════════════════════════════════════
#  CARGA: FUNCIONÁRIOS (dados da Aula 01)
# ════════════════════════════════════════
dados = {
    "nome": ["Ana", "Bruno", "Carlos", "Daniela", "Eduardo"],
    "idade": [23, 35, 29, np.nan, 40],
    "cidade": ["SP", "RJ", "SP", "MG", "RJ"],
    "salario": [3000, 5000, 4000, 3500, np.nan],
    "data_contratacao": ["2020-01-10", "2019-03-15", "2021-07-22", "2018-11-30", "2022-05-10"],
}
df_f = pd.DataFrame(dados)
df_f["idade"]   = df_f["idade"].fillna(df_f["idade"].mean())
df_f["salario"] = df_f["salario"].fillna(df_f["salario"].median())
df_f["salario_anual"]    = df_f["salario"] * 12
df_f["ano_contratacao"]  = pd.to_datetime(df_f["data_contratacao"]).dt.year
df_f["categoria_salario"] = df_f["salario"].apply(
    lambda x: "Alto" if x > 4500 else "Médio" if x > 3000 else "Baixo")
df_f.to_sql("funcionarios", conn, if_exists="append", index=False)
print(f"✅ {len(df_f)} funcionários inseridos")

# ════════════════════════════════════════
#  CARGA: LIVROS (livros.csv da Aula 02)
# ════════════════════════════════════════
df_l = pd.read_csv("livros.csv", sep=";")
df_l = df_l[df_l["paginas"] > 0].drop_duplicates(subset="isbn")
df_l["titulo"] = df_l["titulo"].str.strip()
df_l["autor"]  = df_l["autor"].str.strip()
df_l.to_sql("livros", conn, if_exists="append", index=False)
print(f"✅ {len(df_l)} livros inseridos")

# ════════════════════════════════════════
#  CARGA: BÔNUS (merge da Aula 01)
# ════════════════════════════════════════
cursor.executemany("INSERT INTO bonus VALUES (?, ?)",
    [("Ana", 500), ("Bruno", 1000), ("Carlos", 700),
     ("Daniela", 600), ("Eduardo", 800)])
conn.commit()

# ════════════════════════════════════════
#  EMPRÉSTIMOS
# ════════════════════════════════════════
cursor.executemany("""
    INSERT INTO emprestimos (livro_id, funcionario_id, data_retirada, data_devolucao)
    VALUES (?, ?, ?, ?)
""", [
    (1, 1, "2024-01-10", "2024-01-24"),
    (2, 2, "2024-02-01", None),
    (3, 1, "2024-02-15", "2024-03-01"),
    (4, 3, "2024-03-10", None),
])
conn.commit()

# ════════════════════════════════════════
#  RELATÓRIO FINAL COM JOINS
# ════════════════════════════════════════
df_rel = pd.read_sql_query("""
    SELECT
        f.nome              AS funcionario,
        f.cidade,
        f.salario,
        b.bonus,
        f.salario + b.bonus AS salario_total,
        l.titulo            AS livro_emprestado,
        e.data_retirada,
        CASE
            WHEN e.data_devolucao IS NULL THEN '📖 Em posse'
            ELSE '✅ Devolvido'
        END AS status_emprestimo
    FROM emprestimos e
    JOIN funcionarios f ON e.funcionario_id = f.id
    JOIN livros       l ON e.livro_id       = l.id
    LEFT JOIN bonus   b ON f.nome           = b.nome
    ORDER BY e.data_retirada DESC
""", conn)

print("\n📊 RELATÓRIO DE EMPRÉSTIMOS:")
print(df_rel.to_string(index=False))

# Exporta resultado final
df_rel.to_excel("relatorio_biblioteca.xlsx", index=False)
print("\n📤 relatorio_biblioteca.xlsx gerado!")
conn.close()

Recursos Avançados

🔗 JOIN — Cruzando tabelas

Equivalente ao pd.merge() do Pandas, mas declarativo e mais poderoso.

joins.py
# INNER JOIN — apenas onde há correspondência nos dois lados
df_emprestimos = pd.read_sql_query("""
    SELECT
        l.titulo,
        l.autor,
        f.nome      AS funcionario,
        e.data_retirada,
        e.data_devolucao,
        CASE
            WHEN e.data_devolucao IS NULL THEN 'Em aberto'
            ELSE 'Devolvido'
        END AS status
    FROM emprestimos e
    INNER JOIN livros       l ON e.livro_id       = l.id
    INNER JOIN funcionarios f ON e.funcionario_id = f.id
    ORDER BY e.data_retirada DESC
""", conn)
print(df_emprestimos)

# LEFT JOIN — todos os livros, mesmo sem empréstimo
df_todos = pd.read_sql_query("""
    SELECT l.titulo, COUNT(e.id) AS vezes_emprestado
    FROM livros l
    LEFT JOIN emprestimos e ON e.livro_id = l.id
    GROUP BY l.id
    ORDER BY vezes_emprestado DESC
    LIMIT 10
""", conn)

⚡ Índices — consultas rápidas

Índices são como o sumário de um livro: tornam buscas muito mais rápidas em tabelas grandes.

indices.py
# Cria índice na coluna autor (acelera GROUP BY e WHERE por autor)
cursor.execute("CREATE INDEX IF NOT EXISTS idx_livros_autor ON livros(autor)")

# Índice composto (útil para filtros por ano E autor juntos)
cursor.execute("""
    CREATE INDEX IF NOT EXISTS idx_livros_ano_autor
    ON livros(ano, autor)
""")

conn.commit()

# Ver os índices existentes
cursor.execute("SELECT name, sql FROM sqlite_master WHERE type='index'")
print(cursor.fetchall())

🔐 Transações — tudo ou nada

Uma transação garante que um conjunto de operações seja atômico: se qualquer parte falhar, tudo é revertido.

transacoes.py
try:
    conn.execute("BEGIN")
    
    # Operação 1: adiciona empréstimo
    cursor.execute("""
        INSERT INTO emprestimos (livro_id, funcionario_id, data_retirada)
        VALUES (?, ?, date('now'))
    """, (5, 2))
    
    # Operação 2: atualiza status (exemplo)
    cursor.execute("""
        UPDATE livros SET titulo = trim(titulo) WHERE id = ?
    """, (5,))
    
    conn.commit()   # ✅ ambas tiveram sucesso
    print("Transação concluída!")

except Exception as e:
    conn.rollback()   # ⛔ desfaz tudo
    print(f"Erro, rollback: {e}")

🧾 Row Factory — resultados como dicionário

Por padrão, os resultados são tuplas. Com row_factory, você acessa por nome de coluna.

row_factory.py
conn = sqlite3.connect("biblioteca.db")
conn.row_factory = sqlite3.Row   # ← ativa acesso por nome
cursor = conn.cursor()

cursor.execute("SELECT * FROM livros LIMIT 3")
for row in cursor.fetchall():
    # Agora acessa pelo nome da coluna, não por índice!
    print(f"Título: {row['titulo']} | Autor: {row['autor']} | Páginas: {row['paginas']}")

# Converte facilmente para dicionário
row_dict = dict(row)
print(row_dict)

💾 Backup em memória e exportação

backup.py
# Banco em memória (útil para testes — não persiste)
conn_mem = sqlite3.connect(":memory:")

# Copia o banco do disco para a memória
conn_disco = sqlite3.connect("biblioteca.db")
conn_disco.backup(conn_mem)

# Exporta o SQL completo do banco (dump)
with open("backup_biblioteca.sql", "w", encoding="utf-8") as f:
    for linha in conn_disco.iterdump():
        f.write(linha + "\n")

print("Backup gerado: backup_biblioteca.sql ✅")
